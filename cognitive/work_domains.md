# Work Domains Registry

> **Purpose**: Defines your active work domains for cascade notification by `cognitive-cascade-notifier`
> **Used by**: `cognitive-cascade-notifier` to determine which domains should be notified when cognitive structure updates occur
> **Edit**: Customize the domains to match your actual work areas

---

## Active Work Domains

| Domain ID | Domain Name | Core Responsibility | Examples |
|---|---|---|---|
| domain-1 | [Product / Project Development] | Building and iterating on products/projects | Design, implementation, testing |
| domain-2 | [Knowledge & Research] | Researching and creating knowledge assets | Writing, studying, investigating |
| domain-3 | [Skill System] | Maintaining AI agent Skills, Rules, Agents | Adding Skills, fixing prompts |
| domain-4 | [Content & Communication] | Creating and publishing content | Articles, presentations, reports |
| domain-5 | [Operations] | Running regular operations | Planning, admin, coordination |

---

## Instructions

1. Edit the table above to match your actual work areas
2. Each domain should correspond to a distinct area of your work
3. The `cognitive-cascade-notifier` will notify these domains when principles or L1 docs update
4. Fewer, clearer domains work better than many overlapping ones
