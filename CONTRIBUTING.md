# Contributing to cognitive-os

Thank you for your interest in contributing!

## How to Contribute

### Reporting Issues

- Use GitHub Issues to report bugs or suggest improvements
- For bugs: include your Cursor version, OS, and steps to reproduce
- For feature requests: describe the use case and expected behavior

### Contributing Skills

New `cognitive-*` Skills should follow the format in `.cursor/skills/cognitive-capture-fragment/SKILL.md`:

1. Clear trigger conditions
2. Step-by-step execution protocol
3. Cognitive science annotation (which memory/cognitive mechanism does it implement?)
4. References to relevant L1/L1.5 documents (D0 row)

### Contributing Template Content

Template files in `cognitive/` should be:
- Generic enough for anyone to fill in (no personal content)
- Well-commented with `> Edit this section to...` guidance
- Consistent with the maintenance protocol (C1-C13 conditions)

### Pull Request Process

1. Fork the repository
2. Create a feature branch: `git checkout -b feature/your-feature-name`
3. Make your changes
4. Test: open in Cursor and verify the Skills work as expected
5. Submit a Pull Request with a clear description

## Code of Conduct

Be respectful. Contributions of all kinds are welcome.
